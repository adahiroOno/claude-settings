# 消費面カバレッジマップ(見落とし防止の基盤)

Claude Code がトークンを消費する**すべての面**の分類地図。監査(cost-audit)はこの地図に対して
「環境に存在するものを完全列挙し、全項目をいずれかの面に分類する」ことで網羅性を担保する。
**どの面にも分類できないものは『未分類』としてレポートに必ず載せる**(見落としの構造的排除)。

## 1. 常駐入力 — 毎リクエストのシステムプロンプト/定義部に載る固定費

| 消費面 | 実体の在り処 | チェック項目 |
|---|---|---|
| CLAUDE.md(グローバル/プロジェクト/local、@import 先) | `~/.claude/CLAUDE.md`, `CLAUDE.md`, `CLAUDE.local.md`, `.claude/CLAUDE.md` | C-1 |
| スキルの description(本文はオンデマンド) | `~/.claude/skills/*`, `.claude/skills/*`, プラグイン | F-1 |
| エージェントの description | `~/.claude/agents/*`, `.claude/agents/*` | F-1(同基準) |
| MCP ツール定義(接続中サーバーの全ツール) | `.mcp.json`, `~/.claude.json`, `claude mcp list` | C-2 |
| 出力スタイル本文・--append-system-prompt | `output-styles/*`, `outputStyle` 設定, CLI 引数 | D-2b |
| フックが注入するコンテキスト(SessionStart 等) | 各 settings の `hooks` → スクリプトの stdout | B-2, E-2b |
| 常駐メモリ的な運用(自動追記される作業メモ) | CLAUDE.md への追記, メモリファイル | E-2b |

## 2. 会話に積まれる入力 — 一度入ると以降の全ターンで再送される変動費

| 消費面 | 典型例 | チェック項目 |
|---|---|---|
| Read したファイル内容 | 巨大ファイル・ロックファイル・生成物 | C-3(deny + ガードフック) |
| ツールの標準出力 | テストログ全量・ビルドログ・verbose 出力 | C-6 |
| Web 検索/取得の結果 | 記事全文・検索結果一覧 | C-6 |
| 画像・スクリーンショット | 1枚で数百〜数千トークン | C-6 |
| MCP ツールの出力 | 大きな API レスポンス | C-2, C-6 |
| 会話履歴そのもの(前タスクの残骸) | /clear しない長セッション | C-4 |
| 探索の中間結果 | メイン文脈での横断調査 | C-5(explore 委譲) |

## 3. 出力 — 単価が入力の5倍

| 消費面 | チェック項目 |
|---|---|
| thinking(常時 thinking 化) | D-1 |
| 地の文の冗長性・全文再掲 | D-2(+ terse スタイル D-2b) |
| 出力上限の設定不備(途中切断→リトライ) | D-3 |

## 4. 派生消費 — メインの会話の外側で発生する呼び出し

| 消費面 | チェック項目 |
|---|---|
| サブエージェント(モデル未指定でメイン単価) | A-3 |
| バックグラウンド呼び出し(タイトル生成等) | A-2, A-4 |
| フック内からのモデル呼び出し | G-3 |
| cron / タイマー / CI からのヘッドレス実行 | G-2 |
| 放置プロセス・多重起動 | G-1 |
| 誤発火したスキル(本文読み込み+無関係な指示) | F-4 |

## 5. 係数 — 同じ入力でも実効単価を変えるもの

| 消費面 | チェック項目 |
|---|---|
| キャッシュ無効化(フラグ・動的注入・構成変更・TTL切れ) | B-1〜B-4 |
| モデル単価(メインが Opus のまま等) | A-1 |

---

## 完全列挙の対象(監査フェーズ0で必ず走査するもの)

1. `find ~/.claude -maxdepth 2` および `find .claude -maxdepth 2` の全ファイル・ディレクトリ
2. 各 settings.json(user/project/local/managed)の**全トップレベルキー**と `env` の全変数
3. `claude mcp list` の全サーバー
4. `scan_background.sh` の全検出結果
5. シェル環境変数のうち `ANTHROPIC*` / `CLAUDE*` / `MAX_THINKING*` / `DISABLE_*`

列挙した各項目をこの地図のいずれかの面に対応付ける。対応付けられないもの
(未知の設定キー・見慣れないディレクトリ・新機能の痕跡)は**「未分類(要判断)」として
レポートに記載**し、扱いが確定したらこのファイルへの追記を提案する(地図の自己更新)。
