# handoff — /clear 前の引き継ぎ保存

今のセッションの作業状態を `.claude/handoff.md` に**40行以内**で書き残す。

## 手順

1. `.claude/handoff.md` に次の形式で書く(このセッションの文脈から要約する。ファイル探索はしない):

   ```
   # handoff: <作業の一行要約>(<日付> / <ブランチ名>)
   ## 目的
   <なぜやっているか。1〜2行>
   ## 完了済み
   <確定した成果だけ。ファイルパス付き>
   ## 再開手順(この順で着手)
   1. <最初にやること — コマンド・ファイルパス・関数名まで具体的に>
   2. <その次>
   ## 重要な決定・注意点
   <再開者が知らないと誤る判断だけ>
   ```

2. 引数で補足メモが渡されていれば「重要な決定・注意点」に含める。
3. 書き終えたら1行で報告する:
   「handoff.md を保存しました。/clear 後の新しいセッションに冒頭から自動注入され、1ターン目から再開できます。」

## 判断基準

- **合格基準: このセッションを知らない読み手が、handoff だけを読んで5分以内に手を動かせること。**
  「再開手順」は番号付きで、各行にコマンドかファイルパスを必ず含める(「続きをやる」のような抽象は不可)。
- タイトルの一行要約は必須(新セッション冒頭に注入されたとき、何の作業か即断できるように)。
- 会話の経緯や試行錯誤は書かない — 結論と現在地だけ。40行を超えるなら削る。

## 完了時の後始末(削除 / アーカイブ)

引き継いだ作業が**完了したら handoff.md を片付ける**(陳腐化メモの誤再開を防ぐため)。2通り:

- **既定(削除)**: `.claude/handoff.md` を削除する。
- **アーカイブ運用**: 履歴を残したい/後で「なぜこの設計にしたか」を参照したいときは、削除の代わりに
  `bash ~/.claude/hooks/handoff-archive.sh` を実行する。`.claude/notes/YYYYMMDD-HHMM-<要約>.md`
  に退避される(ファイル名が索引=常駐トークンゼロ)。どちらを既定にするかはユーザーに従う
  (環境変数 `CLAUDE_HANDOFF_ARCHIVE=1` を使っているなら /clear 時の自動スタブも履歴を残す)。

## 並行タスク・過去メモの参照

- **並行タスクを一旦横に置く**: 次に再開する作業だけを `handoff.md`(注入される再開スロット)に置き、
  今は再開しない別タスクは `handoff-archive.sh` で notes/ に退避しておく。
- **過去メモを探す**: `ls -t .claude/notes/`(新しい順=日時+主題の一覧)。キーワードがあれば
  `grep -l "<語>" .claude/notes/*.md` で該当ファイルだけ特定し、その1つだけ読む。全件は読まない。
